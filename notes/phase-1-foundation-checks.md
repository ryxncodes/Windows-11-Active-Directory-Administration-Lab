Issue:
The Windows Server hostname was still the autogenerated install name when AD DS was promoted. After renaming the domain controller to DC01 post-promotion, domain authentication broke with a workstation trust relationship error.

Root cause:
The domain controller identity was changed after AD promotion. Domain controllers rely on consistent hostname, DNS records, SPNs, and AD metadata.

Fix:
Rebuilt the DC from a clean baseline. Renamed the server to DC01 before AD DS promotion, then configured static IP/DNS and promoted it again.

Lesson learned:
Finalize server hostname and static network configuration before promoting a server to a domain controller.
